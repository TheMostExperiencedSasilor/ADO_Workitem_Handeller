# Writing Rules

## SMART Rule

The SMART rule checks whether a draft looks:

- Specific
- Measurable
- Achievable
- Relevant
- Time-bound

The current implementation is a lightweight deterministic check. It is not meant to replace human judgment or AI review. It gives fast feedback before a work item is created.

## Split One Into Three

The split rule turns one broad item into three smaller items:

1. Investigate
2. Implement
3. Validate

This is useful when a User Story or Feature is too large, unclear, or risky.

## Future Rule Ideas

- Require reproduction steps for Bugs.
